200 /*
201 Currently, if audio input is enabled it uses analog input channel 0 and doesn't
202 allow other channels to be used, to avoid interference between the strict audio
203 rate sampling and any control analog conversions which could be in progress when
204 the audio needs to be sampled. This might be solved in future, but workable
205 solutions for control inputs include RC circuits on digital pins, switches, or
206 serial connections (which also may require extra solutions for asynchronous
207 operation). There is a fork of Mozzi on github (posted on the mozzi-users list)
208 which includes a 2-wire async serial adaptation. It would be good eventually to
209 merge that with the main repo.
210 */
