58 /** @ingroup analog
59 Call this in setup() to enable reading analog inputs in the background while audio generating continues.
60 Then call adcReadAllChannels() at the end of each updateControl() and the results for each analog channel will be
61 available by calling adcGetResult(channel_num) next time updateControl() runs.
62 @note This method using adcEnableInterrupt(), adcReadAllChannels() and adcGetResult() is an easy and
63 efficient way to read analog inputs while generating sound with Mozzi. For many
64 sketches, however, simply putting setupFastAnalogRead() in setup() and calling
65 Arduino's usual analogRead() will work fast enough.
66 @note Don't use setupFastAnalogRead() with adcEnableInterrupt().

68 @note In some cases this method can cause glitches which may have to do with the ADC

70 the adcStartConversion(), adcGetResult() methods instead.

88 /** @ingroup analog
89 Prepare an analog input channel by turning off its digital input buffer.
90 This helps to reduce noise, increase analog reading speed, and save power.
91 
92 Here's more detail from http://www.openmusiclabs.com/learning/digital/atmega-adc/:
93 
94 The DIDR (Data Input Disable Register) disconnects the digital inputs from
95 whichever ADC channels you are using. This is important for 2 reasons. First
96 off, an analog input will be floating all over the place, and causing the
97 digital input to constantly toggle high and low. This creates excessive noise
98 near the ADC, and burns extra power. Secondly, the digital input and associated
99 DIDR switch have a capacitance associated with them which will slow down your
100 input signal if you are sampling a highly resistive load.
101 
102 And from the ATmega328p datasheet, p266:
103 
104 When an analog signal is applied to the ADC pin and the digital input from
105 this pin is not needed, this bit should be written logic one to reduce power
106 consumption in the digital input buffer. Note that ADC named_pins ADC7
107 and ADC6 do not have digital input buffers, and therefore do not require
108 Digital Input Disable bits.
109 @param channel_num the analog input channel you wish to use.
110 */
111 void disconnectDigitalIn(byte channel_num);

143 /** @ingroup analog
144 Starts an analog to digital conversion of the voltage at a specified pin. Unlike
145 Arduino's analogRead() function which waits until a conversion is complete before
146 returning, adcStartConversion() only sets the conversion to begin, so you can use
147 the cpu for other things and call for the result later with adcGetResult().
148 This works well in updateControl(), where you can call adcStartConversion() and
149 get the result with adcGetResult() the next time the updateControl()
150 interrupt runs.
151 @param pin is the analog pin number (A0 to A...) or the channel number (0 to ....) to read.
152 @note This is the most audio-friendly way to read analog inputs,
153 but can be messier in your program than the the adcEnableInterrupt(), adcReadAllChannels(), adcGetResult() way.
154 @note Timing: about 1us when used in updateControl() with CONTROL_RATE 64.
155 */
156 void adcStartConversion(unsigned char pin);

167 /** @ingroup analog
168 Waits for the result of the most recent adcStartConversion(). If used as the first function
169 of updateControl(), to receive the result of adcStartConversion() from the end of the last
170 updateControl(), there will probably be no waiting time, as the ADC conversion will
171 have happened in between interrupts. This is a big time-saver, since you don't have to
172 waste time waiting for analogRead() to return (1us here vs 105 us for standard Arduino).
173 @return The resut of the most recent adcStartConversion().
174 @note This is the most audio-friendly way to read analog inputs,
175 but can be messier in your program than the the adcEnableInterrupt(), adcReadAllChannels(), adcGetResult(byte) way.
176 @note Timing: about 1us when used in updateControl() with CONTROL_RATE 64.
177 */

213 /** @ingroup analog
214 Call adcReadAllChannels() in updateControl() and the results for each channel
215 will be available by calling adcGetResult(channel_num) next time updateControl()
216 runs. Reading will happen in the background, using a minimum of processor time
217 and without blocking other code.
218 
219 More detail: adcReadAllChannels() starts an initial conversion which triggers an
220 interrupt when it's complete. The interrupt code stores the result in an array,
221 changes to the next channel and starts another conversion. When all the channels
222 have been sampled, the ISR doesn't start a new conversion, so it doesn't
223 re-trigger itself or use processor time until adcReadAllChannels() is called
224 again. At any time the latest conversion result for each channel is available by
225 calling adcGetResult(channel_num).
226 @note In some cases this method can cause glitches which may have to do with the ADC interrupt
227 interfering with the audio or control interrupts.
228 */
229 void adcReadAllChannels();
