9  *
10  * Audio data, Huffman decoder table, sample rate and bit depth are defined

12  * This file can be generated for a sound file with the
13  * accompanying Python script audio2huff.py
14  *
15  * Invoke with:
16  * python audio2huff.py --sndfile=arduinosnd.wav --hdrfile=sounddata.h --bits=8
17  *
18  * You can resample and dither your audio file with SOX,
19  * e.g. to 8 bits depth @ 10kHz sample rate:
20  * sox fullglory.wav -b 8 -r 10000 arduinosnd.wav
21  *
22  * The header file contains two lengthy arrays:
23  * One is "sounddata" which must fit into Flash RAM (available in total: 16k for ATMega168, 32k for ATMega328)
24  * The other is "huffman" which must fit into SRAM (available in total: 1k for ATMega168, 2k for ATMega328)
25  *
26  * References:
27  * Arduino: http://www.arduino.cc/
